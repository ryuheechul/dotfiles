#+TITLE: Plan: closing the neovim -> emacs gap
#+STARTUP: overview

* Read this first                                                 :meta:

This is a reference menu, not a sprint backlog. Nothing below is meant to
be worked through proactively or systematically - pick up a single item
only when a real, felt need for it actually comes up in daily use. "Let's
knock out Phase 1" is explicitly not the intent, even though the phases
are ordered by effort and risk, roughly smallest first.

This snapshot was written against the neovim config as it stood on
[2026-07-03]. That config keeps evolving on its own, so every reference
below to a specific nvim file, line, or behavior is provisional - re-check
the actual current nvim source before acting on any item, since the thing
being described may have moved, changed, or been removed entirely since
this was written.

* Goal

Make doom emacs feel like [[file:../../../../../nvim/][the neovim config]] for the things that matter -
muscle memory and quality of life - not a full port. Anything without a
daily-use payoff stays in neovim land (see [[*Not porting (deliberately)][not porting]]).

* Phase 1: muscle memory gaps (small, high frequency)

Ported from [[file:../../../../../nvim/lua/plugins/keymaps.lua][keymaps.lua]] unless noted. Each is a few lines in morevil.

** DONE ~gn~ / ~go~ -> jumplist forward/back
nvim: ~<C-I>~ / ~<C-O>~. Both bound to better-jumper in morevil. It turned
out ~go~ already behaved: doom remaps xref-go-back onto
~better-jumper-jump-backward~, which piggybacks evil's jump list (plain
motions like gg/G//search are tracked) - the real gap was only ~gn~,
which was unbound. Now both directions are explicit.

** DONE ~gl~ / ~gr~ / ~gd~ -> definition, references
nvim: ~gd~/~gr~ = lsp definition/references, ~gl~ = ~<C-]>~ via tagfunc.
root cause of "gd does nothing in lua" (2026-07-03): .lua files open in
lua-mode (doom's :lang lua has no +lsp flag) while ext-lang/lua hooked
lsp! only on lua-ts-mode, which nothing routes files to - eglot simply
never attached. fixed by hooking lua-mode too (ext-lang/lua/config.el);
gd needed no binding change (+lookup/definition all along). gl and gr
bound in morevil, shadowing accepted knowingly: gl was evil-lion's
left-align operator (gL survives), gr was doom's +eval:region (gR and
SPC-level eval bindings survive). follow-up (2026-07-04): gl and gd are
NOT the same thing in nvim - gd (vim.lsp.buf.definition) offers a
selection when the server reports several entities (lua-ls: name +
body), which doom's consult-xref picker mirrors correctly, while gl
(C-]) has TAG-jump semantics: follow the "link" to the FIRST match
immediately - and its real heritage is :help, where C-] follows the
help-tag hyperlink under the cursor. so gl is bound (:nm, so doc-ish
motion-state buffers get it too) to +neovim/goto-link (compat/neovim):
a button/Info link when on one, otherwise the first xref definition
with no picker; gd keeps the picker.

** DONE ~Y~ -> yank to end of line
~(setq evil-want-Y-yank-to-eol t)~ in this module's [[file:./init.el][init.el]] - module
init.el files run at doom's early-init stage, before any package (evil
included) loads, which is what the ~evil-want-*~ family requires;
config.el is too late. (evil's default Y = whole line; nvim's = y$.)

** TODO ~j~/~k~ move by visual line when no count
Pairs with the existing ~global-visual-line-mode~:
~(setq evil-respect-visual-line-mode t)~ - same before-evil-loads caveat as ~Y~.

** DONE ~,~ -> repeat last ex command
nvim: ~@:~. Bound to ~evil-ex-repeat~ in morevil. Shadows evil's reverse
f/t repeat (same trade nvim made); ~;~ still repeats forward.

** DONE ~+~ / ~-~ -> increment/decrement number at point
nvim: ~<C-A>~/~<C-X>~. ~evil-numbers/inc-at-pt~ / ~dec-at-pt~ in morevil;
the package was already present transitively but is now declared in
morevil's packages.el so doom sync cannot autoremove it.

** TODO ~SPC RET~ -> run macro q
nvim: ~@q~. Emacs: ~(evil-execute-macro 1 (evil-get-register ?q))~.

** TODO ~SPC j~ / ~SPC k~ -> previous/next page
nvim: ~Hzb~ / ~Lzt~ trick. Emacs equivalent of the same trick:
~evil-window-top~ + ~evil-scroll-line-to-bottom~ (and the mirror), or simply
~evil-scroll-page-up/down~ if the exact cursor placement doesn't matter in
practice.

** DONE ~SPC b t~ -> trim trailing whitespace on buffer
~delete-trailing-whitespace~, bound in morevil (the key was free).

** DONE ~SPC t l~ -> toggle listchars
~whitespace-mode~ toggle (the mode itself is set up in this module).
conflicted with doom's ~doom/toggle-line-numbers~ - resolved by swapping:
listchars takes ~t l~ (nvim muscle memory), line numbers move to the
previously free and more mnemonic ~SPC t n~.

** DONE ~SPC t b~ -> toggle dark/light
Turned out to already exist: ~SPC t b~ was bound to
~toggle-doom-theme-tone~ ([[file:../../my-custom/term-enhance/theme.el][term-enhance/theme.el]]) all along.

** TODO ~SPC w z~ / ~SPC w m~ zoom semantics
Reuse the window-configuration save/restore toggle technique from
term-enhance's ~mux/zoom~ as a general ~SPC w z~; nvim's ~SPC w m~ (maximize
editor + outer mux) can call the ~mux-zoom~ script when ~$TMUX~ is set, like
morevil's C-hjkl tmux handoff does.

** DONE verify ~z=~ (fix spelling picker)
Confirmed working out of the box (2026-07-04) - doom's
~:checkers spell +flyspell~ covers it.

** DONE ~\s~ -> suspend to shell
nvim: ~:suspend~ (leader s). Real suspend is TTY-only: ~suspend-frame~ =
SIGTSTP, back with ~fg~, and correct for ~emacsclient -nw~ too. A GUI
frame has no parent shell (launchd started it), so there ~\s~ keeps
toggling the full-window terminal - whose shell aliases ~fg~ to toggle
back, completing the suspend/fg round trip. Dispatched per keypress on
~display-graphic-p~ in morevil.

** DONE dashboard ~i~ -> open a fresh unnamed buffer
nvim's dashboard offers this as an official menu entry (quick scratch
note, usually never saved; it does NOT enter insert state - that is up
to the user, so neither do we). ported the same way: a real
~+dashboard-menu-sections~ entry in compat/neovim (not morevil: it is a
dashboard behavior, not just a keybinding), with doom's own
~+default/new-buffer~ as the action - no custom command needed. the ~i~
binding must live in the evil normal AUX keymap of
~+dashboard-mode-map~: the menu renderer looks up its displayed key hint
there, and the dashboard's own ~[remap evil-insert] -> ignore~ lives
there too, swallowing ~i~ before any remap in the raw keymap is
consulted (first attempt failed exactly that way). the dashboard also
refuses insert state entirely now (advice on ~evil-insert-state~; an
earlier bug left it stuck in insert because evil-buffer-new's
set-window-buffer does not change the current buffer).

* Phase 2: QoL behaviors

From [[file:../../../../../nvim/lua/boot/misc.lua][boot/misc.lua]].

** DONE autoread
nvim: ~autoread~ (+ shortmess tweak in boot/misc.lua) silently reloads a
file that changed outside the editor (git checkout, formatter, ...)
instead of prompting. emacs: ~global-auto-revert-mode~ +
~global-auto-revert-non-file-buffers~, in this module (doom does not
enable it; measured off).

** DONE help buffers act like ~less~
d/u/f/b scroll in ~help-mode~ now (morevil, next to the older Info-mode
map); verified they were NOT less-like before (d was literally ~ignore~).
nvim's auto-maximize of help windows not ported - doom's popup rules
could via ~set-popup-rule!~ if ever missed.

** DONE spell check on by default
verified doom's :checkers spell already hooks flyspell in prose modes
(org, markdown); added ~flyspell-prog-mode~ on prog-mode for the nvim
global-spell feel, scoped like nvim's treesitter @spell captures:
comments and docstrings but NOT plain strings
(~font-lock-string-face~ dropped from ~flyspell-prog-text-faces~ -
identifiers/paths in strings are all false positives). making it
actually WORK in TUI took two real fixes (2026-07-04): eglot semantic
tokens turn the face property into a LIST that the stock prog-mode
predicate's memq cannot see - overridden in compat/neovim; and the
default faces hide behind a supports-wave display guard that renders as
NOTHING on conservative terminals - replaced via
face-OVERRIDE-spec (a theme beats a defface, nothing beats the override)
with unconditional squiggles that leave text color alone: yellow3 for
misspellings, cyan3 for duplicates, identical in GUI and TUI (those are
the exact RGB the terminal palette gives "yellow"/"cyan"; cyan over the
stock DarkOrange because orange blurs into the misspelling yellow).
doom's flyspell-lazy was wrongly convicted along the way (a confounded
experiment: disabling it overlapped with the predicate fix landing); a
controlled re-test with lazy re-enabled showed TUI flagging works fine
under it, just idle-batched (~1s) - so lazy STAYS enabled, trading a
little lag on a never-urgent signal for keystroke-path headroom.

** DONE save-place
nvim: the LastPlace autocmd in boot/misc.lua reopens every file at the
last cursor position (gitcommit buffers excluded). emacs verification
came back negative: ~save-place-mode~ was OFF in this doom
checkout (the assumption that :config default enables it did not hold) -
enabled in compat/neovim. saveplace's default ignore regexp already
excludes COMMIT_EDITMSG, matching nvim's gitcommit exclusion.

** DONE verify smartcase search
confirmed: ~evil-ex-search-case~ is ~smart~ (2026-07-04).

** DONE yank highlight
~:ui ophints~ (enabled).

** DONE undofile
~:emacs undo~ (enabled).

** DONE clipboard sync
Emacs default + ~:os (tty +osc)~ for OSC 52 (enabled).

** DONE cursorline
Doom hl-line. cursorcolumn intentionally skipped: emacs has no
first-class equivalent and it mattered mostly alongside ~virtualedit=all~,
which can't be fully ported anyway (see [[file:./config.el][config.el]]).

* Not porting (deliberately)

- DAP / neotest / sniprun / conjure / luapad :: debugging and REPL
  workflows differ enough in emacs (dape, ert, org-babel already cover the
  need when it arises); revisit only on real demand, in ~lab~ first per its
  charter.
- UI cosmetics with no muscle-memory weight :: alpha dashboard (doom has
  its own), scrollbar, drop.nvim snow, noice/wilder/dressing (vertico
  replaces the lot), bufferline (workspaces + modeline cover it).
- nvim-ghost / package-info / ChatGPT-era plugins :: stale or superseded.
- nvim's ~t~ -> Telescope buffers mapping :: a mistake on the nvim side,
  not a habit: it's shadowed by other tools there and the buffer picker is
  already reachable via ~SPC b b~/~SPC f b~ on both editors. Removed from
  [[file:../../../../../nvim/lua/plugins/keymaps.lua][keymaps.lua]] on [2026-07-04] instead of porting.
- ~virtualedit=all~ full parity :: impossible at the emacs core level
  (documented in [[file:./config.el][config.el]]); ~onemore~ is the ceiling.
- TTY C-hjkl window motion in raw terminal emacs :: key events genuinely
  don't arrive distinctly (noted in morevil); the ~C-w~/~SPC w~ prefix is
  the accepted fallback, and inside ghostel the terminal-side dispatcher
  already handles it.

* Sequencing

Phase 1 is many independent, commit-sized bindings - knock out in one or
two "emacs - port neovim muscle memory batch N" commits /when actually
picked up/, testing each in both GUI and TTY (the TAB/jk commits showed
GUI/TTY key-symbol splits are the usual trap). Phase 2 next; each item is
isolated. Phase 3 last - each module flip can ride its own commit so a
misbehaving package is a one-line revert.

Reminder: "sequencing" describes the order to prefer /if/ and /when/ work
on this starts, not a schedule. See [[*Read this first][Read this first]] above.
